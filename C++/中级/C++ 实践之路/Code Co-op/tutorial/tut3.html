New Project

The first thing you might want to do with your Version Control System is to set up a project. The project could be brand new, or you might have an existing body of source code that you want to include in the project. In any case, the procedure is simple.

Start by selecting New from the Code Co-op's Project menu.

You will be presented with the following dialog:

In the first field, input the path where your project tree is (or will be) on your machine. If you already have a body of source code, point Code Co-op to the directory where this code resides (its subdirectories will be automatically included in the project). If you're creating a project from scratch, type in the name of the project's top-level directory, not necessarily existing (Code Co-op will create it).

It is possible to have the source tree residing on another machine (a network server). Just provide a network path.

Specify project name--others will be able to join this project using its name.

Unless you are planning on distributing all scripts by hand, accept the default short script names. This way you'll avoid troubles with e-mail programs that don't know how to deal with long file names (surprisingly many don't).

Finally, input your personal data, name, email and (optionally) the phone number. Notice: even if you're not planning on using email for script distribution, you have to input something there (may be fake), because the Dispatcher is using it to identify project members. Notice also that on a satellite you should input the email address of the hub, if you're planning on using e-mail between hubs.

If the project you were creating was brand new, you're done. Otherwise, Code Co-op will selectively add existing files to the project. Since there may be a lot of files in the tree, Code Co-op will let you pick project files by extensions first and by name second. This is the dialog you'll see:

Clicking on an extension adds it or removes it from the list. Once an extension is selected, files with that extension are shown in the right pane. You can then selectively remove them from (or add back to) the list. The example above is typical for a Visual C++ project.

After you've prepared the list of project files, Code Co-op starts adding them to its database. For each new extension it encounters, it will ask you the following question:

There are four basic categories of files: header, source, other text and binary. The most important is the distinction between binary files and other types. Once you assign binary type to a file, you won't be able to see its diffs (which are meaningless, anyway, for non-text files).

You can see both project and non-project files. All project files are initially checked in (their state is In). You can start checking them out, editing and then checking back in (see Code Co-op Help).
