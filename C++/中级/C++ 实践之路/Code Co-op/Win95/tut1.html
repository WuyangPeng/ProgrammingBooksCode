LAN Configuration

Each transport choice leads to a different set of configuration options. Let's start with LAN-only.

On a network, one and only one machine must be designated as a hub, the others will be satellites. You should start by selecting your hub and configuring it, then you can move to satellites.

The major criterion in selecting the hub is the availability. Choose the machine that will be accessible on the LAN most of the time. When the hub is down, satellites cannot communicate. Notice: you don't even have to have any projects on the hub machine, all that's needed is that its Dispatcher is available to distribute scripts.

The configuration of the hub is very simple. There are no more questions to answer. However, before you start configuring the satellites, you have to realize one thing: each Code Co-op machine on a LAN shares one of its directories, which we call its public inbox (it's the PublicInbox directory under the root of the co-op database).

The default share name is CODECOOP, so if your hub machine is called HAL, its public inbox will be visible on the network as \\HAL\CODECOOP. You have to remember that when configuring the satellites, because you will be asked to provide the network path to your hub's public inbox.

By the way, using the same configuration options you can configure a satellite which has no access to a LAN. You'll have to transport scripts on a diskette, or some such, but it will work. The trick is to use a path like a:\ instead of a network path.
