Code Co-op 3.0
Server-less Version Control System

The Differ
For your convenience, Code Co-op comes with a built-in viewer/editor/differ. You start the differ by double-clicking a file in the Code Co-op's File Area, Check-In Area, or Synch Area (You may also select the file and use Open from the Selection menu). Depending on the area and the state of the file, the differ will show a slightly different display.

New: You may check out or un-checkout a file from inside the Differ.

When you open a file:

  • In the File Area, if the file is checked in, you'll be able to view the contents of the file, but not edit it. However, it is possible to check the file out directly from the differ (there is a button on the toolbar and also, when you try to edit the checked-in file, you'll be prompted to check it out. If the file is checked out, the differ becomes an editor. And if you don't like the changes made to the file, you have an options of un-checking it.
  • In the Check-In Area the differ will show two panes. The left pane is the editor and the right pane is the differential display. If the file is not involved in a pending synch, the differential display will highlight the edits you've made since the check-out. If the file is involved in a synch, the differ will help you modify the merge if needed. Synch changes will be highlighted in purple and your changes in shades of yellow and green. The left pane will turn into an editor.
  • In the Synch Area, if the file is not checked out, you'll be able to view the synch changes highlighted in purple. The left pane will show the file as modified by the synch (since it's not checked out, you won't be able to edit it, but again, you have an option of checking it out from the differ). If the file is checked out, the differ will help you perform the merge--the display will then be identical to that from the Check-In Area.
Differential Display

Fig 1. Two-pane merge display. Buttons on the toolbar are: Open, Save, Refresh; Check out, Un-check-out; Cut, Copy, Paste, Delete; Previous Change, Next Change.

The differential display highlights the lines that were involved in a change.

Your edits are highlighted in shades of yellow and green. An added line is bright yellow, a deleted line is dark green with the text crossed out. Lines that have been moved are shown in more subdued colors. When a chunk of text has been modified, it appears as a combination of deleted and added lines.

Synch changes are similarly highlighted in shades of purple.

To jump from one change to the next, click on the forward blue arrow button, or press F4. To jump to previous change, press the backward blue arrow, or Shift-F4. These functions are also available through the Search menu.

During a merge, the Differ distingushes between the edits you've made before unpacking the synch and the ones you've made later. The post-synch additions are highlighted in yellow in the left pane. They remain highlighted across differ sessions (or saves), to help you keep track of your merge progress.

Editing
The differ may also serve as a rudimentary editor. It lacks certain standard editing functions, like undo, search and replace. But it lets you type in and delete text; copy, cut and paste to and from the clipboard, etc. Moreover, it lets you copy text from the right pane to the left pane, including lines marked as deleted. This way, you may undo some changes that came with a synch and with which you don't agree (you have to check the file out before accepting the synch, in order to be able to do the editing).

Although there is no undo, you might always discard the changes made since the last save (these changes are highlighted in yellow) by closing the differ without saving.
