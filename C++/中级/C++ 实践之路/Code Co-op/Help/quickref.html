Code Co-op 3.0
Server-less Version Control System

Quick Reference
Main Window

The main window of Code Co-op contains tabs corresponding to different working areas:
 Files,
 Check-in,
 Mailbox,
 Synch,
 History,
 Projects
(Click on the tab in the picture to jump to its description.)

Directly below the tabs you see the toolbar. Its contents is context-sensitive. Different buttons are visible in different areas. Some buttons are enabled only when there is a selection in the main window. All commands in the toolbar are also available from the menus.

You will probably use the file area most - let's see what you can do in it.

File area

Toolbar buttons in the file area correspond to the following commands, from left to right,

  • Check out selected files.
  • Check out all files in the current folder.
  • Un-CheckOut (restore selected files to their pre-check-out state).
  • Create a brand new file in the project.
  • Add selected files and folders to the project.
  • View file (also available by double-clicking on a file).
  • Delete selected file(s).
  • Cut selected files (in preparation for move)
  • Paste files (to finish move)
  • Refresh
  • Folder-up (folder-down is available by double-clicking on a folder).

Some buttons are enabled only when you make a selection.

In the File area, you can view files and folders that are in the project tree (not necessarily added to the project). This area corresponds to the file-system view of the project tree. Remember, files in the project can be also accessed directly from other applications, e.g., your editor or development environment.

Check-in area
This is where all the checked-out, deleted and new files go before being checked-in. Double clicking on a file in this area shows the differences between the current state of the file and the state from before the check-out. You can see what changes you've made so far.


The first button, "check-in all", is the most used one. In some situations, when you want to do a partial check-in, you can use the second, "check-in selected", button. Using the third button you can Un-CheckOut a file and undo all the edits you've made since the check-out.

Mailbox
The Mailbox area is where you can see all the synch scripts--incoming and outgoing. This is where you do the synchronization by unpacking scripts. Scripts that are ready to be unpacked have distinct icons. This area corresponds to the OutBox and InBox folders in the file system.

The first button is used to unpack the message(s) marked as "Next" and perform the synch. If you hold the mouse cursor over the depressed area in the toolbar, you'll see the synch comment in full, as a tooltip (in case it doesn't fit in the depression).

Synch Area
This is the area where you can examine the results of a synchronization. You can look at the files that were changed by the synch. Double clicking shows the differences between the new version and the old version. This is where you perform your code reviews. Once you are done reviewing the changes, you accept the synch (first button) and continue.

If you hold the mouse cursor over the depressed area in the toolbar, you'll see the synch comment in full, as a tooltip (in case it doesn't fit in the depression).

"Accept Synch All" is the most common operation in the synch area--that's the first button. "Accept Synch Selected" follows it. Notice also the check-out buttons. You can do the "pre-emptive check-out" if you don't like some of the proposed changes and make some edits on the spot. The left pane of the Differ serves as an editor (remember to "refresh" the view in the Differ after you've checked out a file).

History
This is the repository of all executed scripts in the order in which they were applied to the project. Your scripts, generated during check-ins, are also shown here.

You can type in file names (with wildcards) in the edit field of the combo box to view the history of selected files. An even better method is to go to the Files area, select some files and choose "View History" from the context menu (right button click). This will bring you to the History area and show you all the scripts that modified these files.

In the History area you also have access to the selection menu that lets you view and restore previous versions of files or the entire project.

Projects

This is the list of all projects on your machine. You can navigate between projects simply by double-clicking on them. Notice that projects with unprocessed incoming scripts are marked with a wave.
