Dear Users,

 

This application is intended as a visual representation of 2D transformations.

The functionality within the workspace, such as picking, undo, redo, and trace,

is only a representation of 1 way to handle these things. Optimization was not

considered a factor in its development. We hope that both the application & the

code behind it help in your journey towards learning the basics behind math used

in game programming.

*Recommended System Requirements*

*Notes*

1) The different actions that the ToolBar Icons represent are displayed on the

   status bar.

2) The Matrix Dialog Box does not check for bad matrices, it is only checked

   for appropriate data in each field

 

3) If you would like to see how the matrices are concatenated & applied, open the following

   with your compiler if you have one, wordpad, or notepad:

            GraphicsUtil.h

            GraphicsUtil.cpp

            Entity.h

            Entity.cpp
