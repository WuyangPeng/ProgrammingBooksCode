About the Game Programming Gems 2 CD-ROM

This CD contains source code and demos which demonstrate the techniques described in the book.  Every attempt has been made to ensure that the source code is bug-free and will compile easily on multiple platforms.  Please refer to the websites http://www.GameProgrammingGems.com and http://www.satori.org/ for errata and updates.

CONTENTS

SourceCode: The source code and demos contained on this CD are contained in a hierarchy of subdirectories based on section name and Gem title/author.  Source code and listings from the book are included.  At each author's discretion a complete demo is sometimes included.  Windows demos were compiled using Microsoft Visual C++ 6.0 and are indicated by a .dsw file.  Executables for each demo are included on the CD, and you can recompile them using the .dsw file and/or Makefile.

ColorImages: For your amusement we've placed TIFF files (RGB flavor) of the book's color plates on the CD as well.  Special thanks to the authors of these fabulous images.

glSetup: Included on this CD you will find the glSetup "monolithic" v1.0.0.121 beta distribution, courtesy of Chris Hecker and GLSetup Incorporated (http://www.glsetup.com/).  glSetup is a program which automates the installation of Windows OpenGL drivers.  You can use this program to install the newest drivers for your 3D graphics card as of April 12, 2001 (the date of this program's creation).  You can also visit the glSetup website and download the latest version for free.

GLUT: In this directory you will also find the GLUT v3.8 beta distribution, courtesy of Mark J. Kilgard.  The files in the GLUT directory include the source distribution, datafile distribution, Win32 library and header file, and the manual pages.  You can compile versions for your own system directly from these distributions, or visit the official GLUT website at http://reality.sgi.com/mjk/glut3/glut3.html.   For Windows-specific information, please bisit Nate Robins' website at http://www.xmission.com/~nate/glut.html.

DirectX8: If you're running on Windows, you're very likely using the DirectX API.  For your convenience this directory holds version 8.0a of the DirectX SDK.

OTHER USEFUL PACKAGES

The Mesa 3D Graphics Library: Mesa is a 3D graphics library with an API which is very similar to that of OpenGL.  Actually, it's basically a drop-in replacement for OpenGL.  If for some reason you can't get your OpenGL drivers to work under Windows or Linux, check out Mesa!  Mesa is located at http://www.mesa3d.org/.

WinZip: WinZip is an archive utility for Windows.  You can use it to unpack the GLUT ZIP files on the CD.  The evaluation version can be download for free at http://www.winzip.com/.

Paint Shop Pro: Paint Shop Pro is a great program which you can use to edit images, photos, Web graphics, etc.  The evaluation version can be downloaded for free at http://www.jasc.com/.

SmartDraw: SmartDraw is a diagram editor which allows you to create charts and move many diagram formats between programs seamlessly.  It is a life-saver if you ever need a quick diagram for your slide presentation or want to get some line art copied out into a bitmap.  You can download the free trial program at http://www.smartdraw.com/.

StarOffice: StarOffice is a free suite of tools very similar to Microsoft Office.  The suite is available for Solaris, Windows, Linux and OS/2 and interoperates with Microsoft formats very well.  You can get the tools from Sun Microsystems via their website at http://www.sun.com/.

Acrobat Reader: To read the PDF included with the GLUT 3.8 distribution you will need a copy of the Adobe Acrobat Reader.  It is a very handy way to distribute typeset files over the net, and it's available for Windows, Mac, OS/2, Palm, and a variety of Unix flavors.  You can download Acrobat Reader via http://www.adobe.com/.

RELATED WEBSITES

If you're reading this, you're probably either a professional game developer, want to be a professional game developer, or you're interested in writing games just for fun.   There are many, many web resources for game developers.  Here are a few of the best that you should know about:

GDConf: The Game Developers Conference, it is currently held every year in San Jose, California (USA).  Their website is at http://www.gdconf.com/.

GDMag: Game Developer Magazine, arguably the game development industry's best magazine for developers.  It is published monthly, and you can may be able to get it for free in the U.S. if you're a professional developer.  Source code to accompany the magazine articles is contained on their website, at http://www.gdmag.com/.

Gamasutra: Gamasutra is the web equivalent of GDMag.  It publishes some of the articles from GDMag as well as articles written specifically for the website.   Gamasutra also includes industry news, community chat areas, and a variety of other useful services.  Find Gamasutra at http://www.gamasutra.com/.

International Game Developers Association: The IGDA is a non-profit association dedicated to linking game developers together around the globe, and encouraging the sharing of information among them.  Their website is http://www.igda.org/.

There are many other great websites, print publications, and conferences popping up around the world to support our industry.  Be sure to support your local organizations!

EDITOR'S NOTES

The intent of having this code available on CD is multifold.  A lot of the source code included here can be used in your games right now!  Some of the other source code can be browsed to learn particular techniques, or cannibalized and tweaked for use in utilities and libraries.  Whatever you use it for, I hope you enjoy it and learn from it.

The source code and demos worked properly as of June 2001, and I'm going to put a bit of information here about the test machine to give you a sense of 1) a working configuration, and 2) what technology was like in the year 2001!

All code was tested on komu.satori.org, a PentiumII 450MHz (256MB) machine with an NVIDIA 32MB GeForce2 GTS (25 Mverts/second) graphics card.  On the Windows side it runs WinME with DirectX 8.0a, GLUT 3.8 and Microsoft Visual C++ 6.0.  On the Linux side it is mostly the Debian GNU/Linux 2.2 "potato" release, with kernel version 2.2.15, XFree86 4.0, the NVIDIA Beta OpenGL driver, GLUT 3.7, Mesa 3.2, GCC 2.95.2, and glibc 2.1.2.  In practice, both of these configurations were very stable for OpenGL development and 3D performance was excellent.

During this project, the SmartDraw and Paint Shop Pro utilities were invaluable to me, so I have included links to their websites above.  Emacs and VI were also used pretty heavily for code editing, but you probably know where to find those if you like them!  The game-related websites listed are very useful and many of the Gem authors have been published in them or have presented at the conference.  Check them out!

SYSTEM REQUIREMENTS

Windows: Intel® Pentium®-series, AMD Athlon or newer processor recommended.   Win95 (32MB RAM) or WinNT4 (64MB RAM) or later required.  3D graphics card recommended for optimal performance.  DirectX 8 and GLUT 3.7 or newer are also required. 

Linux: Intel® Pentium®-series, AMD Athlon or newer processor recommended.   Linux kernel 2.2.x or later required.  32MB RAM recommended.  3D graphics card recommended for optimal performance.  XFree86 4.0, GLUT 3.7, OpenGL driver, glibc 2.1 or newer are also required.  Mesa can be used in place of 3D hardware support.
